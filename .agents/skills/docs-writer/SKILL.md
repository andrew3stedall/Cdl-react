# Docs Writer Skill

## Purpose

Keep documentation aligned with planning and implementation.

## Documentation targets

- `docs/features/active/` for active delivery scope.
- `docs/wiki/` for stable summaries and developer reference pages.
- `docs/architecture/` for ADRs.
- `docs/releases/` for release evidence.
- `CHANGELOG.md` for visible change history.

## Rules

- Update docs with meaningful code or planning changes.
- Keep feature docs aligned with wiki summaries.
- Record validation status honestly.
- Do not move active feature docs to completed folders unless the release workflow calls for it.
- Prefer concise, linked docs over disconnected notes.
