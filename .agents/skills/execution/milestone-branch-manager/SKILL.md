# Milestone Branch Manager Skill

## Purpose

Keep milestone branches scoped, reviewable, and ready for PR review.

## Rules

- Branch from the current default branch unless the milestone plan says otherwise.
- Use a branch name that identifies the milestone or issue group.
- Keep unrelated changes out of the branch.
- Compare the branch against `main` before opening or updating a PR.
- Include docs, changelog, validation notes, data impact, operational impact, and follow-up work in the PR body.
- Do not merge unless merge rules in `AGENTS.md` are satisfied.
