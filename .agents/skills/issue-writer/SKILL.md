# Issue Writer Skill

## Purpose

Create developer-ready GitHub issues that can be implemented without repeated clarification.

## Required issue sections

Every issue should include:

- Purpose.
- Background.
- Scope.
- Out of scope.
- Acceptance criteria.
- Dependencies.
- Data impact.
- UX impact.
- API/backend impact.
- Frontend impact.
- Test expectations.
- Documentation required.
- Risks.
- Follow-up work.

## CDL React rules

- Link to the active feature document or ADR that owns the work.
- Keep issues small enough to review, but group them inside coherent milestones.
- State whether the issue touches PostgreSQL, repository mode, migrations, imports, GCP, frontend routes, or shared contracts.
- Do not create implementation scope that contradicts active docs.
