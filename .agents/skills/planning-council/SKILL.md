# Planning Council Skill

## Purpose

Use this skill to convert rough CDL React intent into structured delivery artefacts.

## Mode

Planning only. Do not implement application code or change runtime behaviour.

## Role loop

Run these perspectives in order:

1. Executive Sponsor: confirm alignment with the product goal.
2. Product: define user value, MVP relevance, and priority.
3. Requirements / Business Analyst: decompose work into epics, milestones, issues, dependencies, and success criteria.
4. UX: define user, admin, app, and developer flows.
5. Metrics: define scoring, analytics, dashboard, FDR, prediction, or evaluation requirements where relevant.
6. Data: define PostgreSQL, repository, migration, import, quality, and cache impacts.
7. Platform: define CI/CD, environment, and delivery constraints.
8. Developer Feasibility: provide implementation feasibility notes only.
9. Full-Stack Developer: identify likely backend/frontend implementation shape without coding.
10. Docs Writer: define documentation updates required for each issue.
11. Integration: identify cross-feature contract impacts.
12. Reviewer: identify gaps, risks, and merge-readiness blockers.

## Output

Produce a planning report with summary, alignment decision, proposed milestones, proposed epics, issue list, dependency order, risks, documentation updates, labels, and next human decisions.

## Ready issue standard

Each issue must include goal, background, scope, out-of-scope items, user/system flow, acceptance criteria, technical notes, data notes, UX notes, infrastructure notes, dependencies, test expectations, documentation required, risk rating, and open questions.
