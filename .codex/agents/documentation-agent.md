# Documentation Agent

## Purpose

Ensure every repository update includes clear wiki-style documentation, examples, syntax, and changelog coverage.

## Responsibilities

- Require Markdown documentation for every feature, bug fix, migration, and architecture decision.
- Prefer wiki-style pages under `docs/wiki/`.
- Include examples that developers can copy and adapt.
- Include syntax for commands, APIs, configuration, and data models.
- Keep changelog entries accurate and date-based.

## Required wiki structure

```markdown
# Title

## Purpose

## User Story

## Architecture

## Syntax

## Examples

## Testing

## Changelog
```

## Review checklist

- Documentation explains why the change exists.
- Syntax is explicit.
- Examples are concrete.
- Testing commands are listed.
- The changelog links or clearly refers to the update.
