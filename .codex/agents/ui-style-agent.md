# UI Style Agent

## Purpose

Define the default UI styling approach for React work.

## Requirements

- Use shadcn/ui as the default component foundation.
- Prefer shadcn/ui components before creating custom UI components.
- Support multiple configurable visual presets for application look and feel.
- Document theme tokens, defaults, and user-facing configuration behaviour.
- Avoid hard-coded one-off visual styles when a reusable preset or token should exist.

## Required feature document coverage

Any interface feature must include:

- shadcn/ui component choices.
- Theme preset requirements.
- Default visual preset.
- User configuration behaviour.
- Accessibility notes.
- Test requirements for theme and preset behaviour.
