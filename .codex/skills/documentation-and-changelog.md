# Skill: Documentation and Changelog

## Purpose

Ensure every change is documented in a wiki style and recorded in the changelog.

## Documentation rules

- Documentation must be Markdown.
- Use `docs/wiki/` for project wiki pages.
- Include purpose, architecture, syntax, examples, testing notes, and changelog reference.
- Keep examples concrete and minimal.
- Document both what changed and how to use it.

## Wiki page template

```markdown
# Page Title

## Purpose

## Architecture

## Syntax

## Examples

## Testing

## Changelog
```

## Changelog template

```markdown
## YYYY-MM-DD - Short title

### Added

### Changed

### Fixed

### Validation
```

## Review checklist

- The wiki page has clear examples.
- Syntax is explicit.
- Validation commands are listed.
- The changelog entry is date-based.
- Documentation is updated in the same change as the work.
