# Skill: Feature Specification

## Purpose

Create implementation-ready Markdown specifications before code is written.

## Inputs

- Feature goal.
- User roles.
- Required API behavior.
- Required UI behavior.
- Data and validation rules.
- Testing expectations.

## Process

1. Define the problem and expected outcome.
2. Write user stories and acceptance criteria.
3. Define API contracts and data models.
4. Identify React components, hooks, Python services, repositories, and Pydantic models.
5. Define unit and integration test coverage.
6. Add documentation and changelog requirements.

## Output template

```markdown
# Feature: Name

## Purpose

## User Stories

## Acceptance Criteria

## API Contract

## Data Models

## Frontend Design

## Backend Design

## Testing Plan

## Documentation Plan

## Changelog Plan
```
