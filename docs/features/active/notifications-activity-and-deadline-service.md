# Notifications, Activity, and Deadline Service

## Purpose

Define notifications, activity feeds, audit separation, deadline reminders, watchlist alerts, draft events, and approval queue integration.

## Status

Checkpoint 3 implementation contracts are in progress on `checkpoint-3-squad-movement`.

## Business Rules

- Notifications, activity events, and audit records are separate.
- Activity scopes: league public, team private, negotiation private, commissioner private, system admin.
- In-app notifications come first.
- Email is optional for critical events.
- Each actionable notification has one primary action URL.
- Deadline reminders are generated by a central service.
- Watchlist notifications focus on availability/status changes, not every points change.
- Draft events are durable, not only live messages.

## API Requirements

- List notifications.
- Mark read/unread.
- Get activity feed by scope.
- Calculate upcoming deadlines.
- Send or queue deadline reminders.
- Get approval queue notification counts.
- Manage simple notification preferences.

## React Requirements

- Notification centre.
- Unread counts.
- Activity feed components.
- Approval queue entry point.
- Deadline cards and reminders.
- Watchlist alert display.

## Data Access Requirements

- Notifications target users.
- Activity events respect visibility scope.
- Audit remains immutable/accountability-focused.
- Avoid duplicate reminders for the same deadline window.

## Acceptance Criteria

- A lineup deadline warning links to team selection.
- A transfer approval notification links to approval queue.
- Private negotiation events do not appear in public league feed.
- Watchlist player availability changes can notify the manager.
