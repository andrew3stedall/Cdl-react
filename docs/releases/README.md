# Release Documentation

This folder captures release-level documentation for completed feature work.

## File naming

Use one file per release:

- `v0.1.0.md`
- `v1.0.0.md`

## Required sections

Each release document should include:

- Release Type
- Completed Features
- User-Facing Changes
- API Changes
- UI and Theme Preset Changes
- Data and Migration Changes
- Testing Evidence
- Known Gaps

## Linkage

Every completed feature document under `docs/features/completed/{major|minor}/vX.Y.Z/` must be referenced in `docs/releases/vX.Y.Z.md`.
