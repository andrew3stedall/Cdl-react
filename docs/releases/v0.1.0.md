# Release v0.1.0

## Release Type

Minor.

## Completed Features

No completed feature documents yet. Active feature documents remain active until validation and release completion.

## User-Facing Changes

- Initial React shell placeholder.
- Added authentication and protected-route foundation behaviour.
- Added a searchable rules knowledge base surface with stable rule section links.
- Added responsive shared application shell navigation and selectable visual presets.
- Added a League route with current fixtures, upcoming fixtures, standings, knockout context, and head-to-head context.
- Added a Squad Management route with squad summary, scouting, interests, proposed trades, and shared player detail.
- Added a Team Selection route with lineup editing, chip state indicators, bench/reserve panels, and fixture/table summaries.

## Planning and Migration Changes

- Expanded the legacy migration feature index with reviewed legacy entry points, mapped active feature documents, implementation status, migration risks, missing-feature candidates, and validation requirements.
- Added wiki documentation for the legacy migration feature index.
- Added documentation tests to prevent reviewed legacy behaviours, active feature mappings, missing-feature candidates, and risk tracking from being dropped silently.

## API Changes

- Added health endpoint.
- Added initial theme preset contract endpoint.
- Added shared Pydantic contract models.
- Added login, session status, and logout endpoints.
- Added rules list, category filter, search, and detail endpoints under `/api/rules`.
- Added league fixture, table, knockout, and head-to-head endpoints under `/api/league`.
- Added squad management endpoints under `/api/squad`, `/api/scouting`, `/api/interests`, and `/api/trades`.
- Added team selection endpoints:
  - `GET /api/team-selection`
  - `PUT /api/team-selection/lineup`
  - `PUT /api/team-selection/chips/{chip_id}`
  - `GET /api/team-selection/fixtures-summary`
- Added user preference endpoints:
  - `GET /api/me/preferences`
  - `PUT /api/me/preferences`

## UI and Theme Preset Changes

- Added initial `classic`, `dark`, and `compact` visual preset names.
- Added frontend auth helpers that can be consumed by the future application shell.
- Added rules page rendering that preserves rule readability across theme preset selection.
- Added route-aware active navigation state for Squad Management, Team Selection, League, Rules, Dashboard, FDR, and Scouting.
- Added League route rendering inside the shared authenticated shell.
- Added Squad Management route rendering inside the shared authenticated shell.
- Added Team Selection route rendering inside the shared authenticated shell.
- Added visual preset tokens for colour, density, radius, typography scale, and chart palette hooks.
- Added CSS variable application for shell-level theme consistency.

## Data and Migration Changes

- No database migrations.
- Authentication uses an in-memory development repository pending production user/session storage.
- Rules use structured in-code content pending Markdown, database, or CMS-backed storage.
- League uses an in-memory repository pending migration of legacy fixture/result storage.
- Squad management uses an in-memory repository pending migration of legacy squad, interest, and trade storage.
- Team selection uses an in-memory repository pending migration of legacy lineup and chip storage.
- Legacy migration index now tracks dashboard and FDR as active planning gaps until those features are implemented.
- Added in-memory user preference repository as a documented fallback until persistent user preference storage is introduced.

## Testing Evidence

- Backend service tests cover login, session, rules, league, preferences, squad, and team selection workflows.
- Backend API tests cover rules, league, preferences, squad, and team selection endpoints.
- Documentation tests cover legacy migration entry-point mapping, active feature mapping, missing-feature candidates, and migration risk visibility.
- Frontend tests cover protected routes, rules, shell navigation, League rendering, Squad Management interactions, and Team Selection interactions.

## Known Gaps

- Production user and session storage are not implemented yet.
- Rule admin editing and persisted rule version history are planned but not implemented.
- League fixture/result data is in-memory until legacy data access is migrated.
- Squad management data is in-memory until legacy squad, interest, and trade data access is migrated.
- Team selection data is in-memory until legacy lineup and chip data access is migrated.
- Dashboard and FDR feature endpoints are not implemented yet.
- Feature documents are intentionally not moved to completed until validation passes.
- The legacy migration feature index remains active until legacy migration coverage is complete.
- The preference repository is process-local until the backend data platform introduces persistent user preference storage.
